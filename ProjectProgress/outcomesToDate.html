Front End (Successful Implementation)

We already completed the user interface for the login account feature and the navigation bar feature in Google Chrome. See the screenshots below. We changed our background color differently with mock up, because different pages are done by different group members, so we wanted to combine different pages together, we chose a simple background color so that we could visually combine all the pages together and it was easy for the user to look.

Figure: MVF1 - Account Management for Registration(Google Chrome)

Successful implementation: When a user wants to register in our online store, they need to fill in First name, Last name, Email, Phone number, Password. Also, registration requires all fields must be filled in, the mobile phone number must be in the correct format, and the password must be at least 6 digits. If each field is in the correct form and then they can successfully register.
Figure: MVF1 - Account Management for User login

Successful implementation: After user registration, there are some pages such as checkout, create new blogs and contact us page requiring the user to log in with Email and Password, and then they can successfully access our website. Users can easily log in or log out on each page.
Figure: MVF2 - Online Gaming page (Google Chrome)

Successful implementation: We create a simple game similar to the Greedy snake game. The game can successfully play, each user only can get one chance, users can get the corresponding coupon code by playing online games.

Figure: MVF3 - Blog (Google Chrome)

Successful implementation: We have successfully created our blog page, users can visit our beautiful blog page without log in, when they click on read more, they can see each comment by different customers, also more description of each blog.
Figure: MVF 3.1 - create a new blog page(Google Chrome)

Successful implementation: We have successfully set a new blog for users, but before they insert a blog, they must login and then start implementing a new blog, they can share their ideas or leave a comment under each blog. This ensures that users can communicate with each other and share funny ideas.

Figure: MVF4 - Checkout for Purchase Products (Google Chrome)

Successful implementation: We have successfully created a Checkout after the user selects the product, but not all products provide coupon code, if users win the games and get coupon code, they can enter the coupon code when they purchase, they also need to fill out the information in a correct format and then press checkout, they order will be purchase successfully.
Figure: MVF5 - Ranking List

Successful implementation: We have successfully set a ranking list for users, after they play games on our website, If they get top ten of the ranking list, the name will be on the left side of the game box, also the name appears under the RULES. We will clear our ranking list every day. Each user has only one chance, so they will play very seriously to get high scores.
Figure: MVF5.1 - Music (Google Chrome)

Successful implementation: We have successfully inserted music at top of the game, so that users can enjoy the wonderful music while playing the game. The sound of the music can also be adjusted up or down, but if they don’t want to listen to the music, we also provide a pause button so that they can also play the game quietly.
Figure: MVF6 - Display Goods for product selling

Successful implementation: We have successfully created our online game store, we have three pages of game, PlayStation 4, PC, Xbox, each page contains many different games, so that users can have many choices to choose what they want. They can press the Checkout button to purchase their games.
